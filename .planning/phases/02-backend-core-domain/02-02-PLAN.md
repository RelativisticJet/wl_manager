Create bin/wl_trash.py (~250 lines) exporting 8 public functions for trash management.

**Imports:**
```python
import json
import os
import shutil
from datetime import datetime, timedelta
from pathlib import Path

    OWN_LOOKUPS, TRASH_DIR, TRASH_RETENTION_DAYS, TRASH_CONFIG_FILE,
    sanitize_text
)
from wl_validation import sanitize_text
```

**Public API:**

1. `get_trash_dir() -> str` — Return absolute path to trash directory. Create if missing.

2. `read_trash_config() -> Dict` — Read trash configuration JSON file. Return dict with keys: {items: list[...], metadata: {...}} or empty dict on error.

3. `write_trash_config(config: Dict) -> None` — Write trash configuration JSON file atomically. Raise OSError on permission denied.

4. `move_to_trash(item_type: str, name: str, user: str, comment: str, app_context: str = "", version: str = "", detection_rule: str = "") -> str` — Move CSV or rule to trash. Create trash item with unique ID, store metadata, return trash_id. Raise OSError on file operations failure.

   Parameters:
   - item_type: "csv" or "rule"
   - name: CSV filename or rule name
   - user: username performing delete
   - comment: reason for delete
   - app_context: optional app context
   - version: optional version for CSV reverts
   - detection_rule: optional detection rule for CSV context

   Returns: trash_id (UUID)

5. `list_trash() -> Tuple[List[Dict], str]` — List all trash items. Return (items_list, error_string). Items sorted by timestamp (newest first).

   Item dict keys: {trash_id, item_type, name, user, comment, timestamp, ...}

6. `restore_from_trash(trash_id: str) -> Tuple[Dict, str]` — Restore item from trash by ID. Return (restored_item, error_string). Move file back to original location, update config.

7. `purge_trash_item(trash_id: str) -> Tuple[bool, str]` — Permanently delete item from trash. Return (success, error_string). Delete file and remove from config.

8. `auto_cleanup_trash() -> int` — Automatically purge trash items older than TRASH_RETENTION_DAYS. Return count of items purged.

**Error Handling:**
- move_to_trash: raise OSError if file operations fail (copy, move)
- list_trash, restore_from_trash, purge_trash_item, auto_cleanup_trash: return (data, error_msg) tuples instead of raising
- read_trash_config: silently return empty dict on missing/invalid JSON

**Lock Management:**
- No explicit locking (trash is low-contention); approval queue (Phase 3) handles concurrent approvals

**Type Hints:**
All functions fully type-hinted.

```python
]
```
