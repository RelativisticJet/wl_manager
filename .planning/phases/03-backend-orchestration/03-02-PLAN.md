**Public API:**

1. `get_approval_queue_path() -> str` — Return absolute path to approval queue JSON file (APPROVAL_QUEUE_FILE from constants).

2. `read_approval_queue() -> List[Dict]` — Read approval queue from JSON file. Return list of queue items. On file missing or JSON error, log warning and return empty list.

   Queue item schema:
   ```python
   {
     "request_id": "uuid-string",
     "status": "pending|approved|rejected|cancelled|expired|auto_cancelled|failed",
     "timestamp": int (unix timestamp),
     "analyst": "username",
     "action_type": "save_csv|add_rule|remove_rule|...",
     "detection_rule": "rule name",
     "csv_file": "filename.csv",
     "app_context": "context",
     "payload": {...},  # Original request data
     "pending_highlight": {...},  # Highlight state for UI
     "resolved_by": "admin_user or 'system'",  # Only if not pending
     "resolved_at": int (unix timestamp),  # Only if not pending
     "rejection_reason": "string",  # Only if rejected
     "cancellation_reason": "string",  # Only if cancelled
     "failure_reason": "string"  # Only if failed
   }
   ```

3. `write_approval_queue(queue: List[Dict]) -> Tuple[bool, str]` — Write queue to JSON file atomically. Return (success, error_msg).

4. `submit_approval_request(action_type: str, analyst: str, detection_rule: str, csv_file: str, app_context: str = "", payload: Dict = None, pending_highlight: Dict = None) -> Tuple[str, str]` — Submit a new approval request. Return (request_id, error_msg).

   Steps:
   1. Generate UUID for request_id
   2. Create queue item with status="pending", current timestamp
   3. Acquire _approval_queue_lock
   4. Read existing queue
   5. Append new item
   6. Write queue atomically
   7. Release lock
   8. Return (request_id, "")

   On error: return ("", error_msg)

5. `get_approval_request(request_id: str) -> Tuple[Dict, str]` — Get a specific approval request by ID. Return (item_dict, error_msg).

   On not found: return ({}, "Request not found")

6. `process_approval_decision(request_id: str, decision: str, admin_user: str, rejection_reason: str = "", cancellation_reason: str = "", admin_comment: str = "") -> Tuple[bool, str]` — Process approve/reject/cancel decision on a request. Return (success, error_msg).

   Parameters:
   - decision: "approve", "reject", or "cancel"
   - rejection_reason, cancellation_reason: sanitized text (max 500 chars)
   - admin_comment: sanitized text

   State machine transitions:
   - pending + decision="reject" → status="rejected", resolved_by=admin_user, rejection_reason
   - pending + decision="cancel" → status="cancelled", resolved_by=admin_user, cancellation_reason
   - pending + decision="approve" → (call replay_approved_request) → status="approved" or "failed"

   Steps (for all decisions):
   1. Acquire _approval_queue_lock
   2. Read queue
   3. Find request by request_id
   4. Validate: status="pending", analyst != admin_user (for approve), decision valid
   5. Update item: status, resolved_by, resolved_at
   6. Expire old requests (prune history)
   7. Write queue atomically
   8. Release lock
   9. Build audit event: action="request_[status]", include original request details
   10. Post audit event (if not None session_key; optional param)
   11. Return (True, "")

   On error: return (False, error_msg)

7. `auto_cancel_conflicting_requests(csv_file: str, action_type: str) -> Tuple[int, str]` — Auto-cancel pending requests that conflict with a destructive action (delete CSV/rule). Return (count_cancelled, error_msg).

   Conflicting actions:
   - If action_type="remove_csv" and csv_file="X.csv" → cancel all pending requests with csv_file="X.csv"
   - If action_type="remove_rule" and detection_rule="R" → cancel all pending requests with detection_rule="R"

   Steps:
   1. Acquire _approval_queue_lock
   2. Read queue
   3. Find all pending items with matching csv_file or detection_rule
   4. For each: status="auto_cancelled", resolved_by="system", resolved_at=now, rejection_reason="CSV/rule deleted by approved request"
   5. Write queue
   6. Release lock
   7. Return (count_cancelled, "")

8. `expire_pending_approvals() -> Tuple[List[Dict], str]` — Auto-reject pending requests older than APPROVAL_EXPIRY_DAYS. Prune resolved entries older than 30 days. Cap resolved history at MAX_RESOLVED_HISTORY. Return (updated_queue, error_msg).

   Steps:
   1. Read queue
   2. For each pending item: if age > APPROVAL_EXPIRY_DAYS, status="expired", resolved_by="system"
   3. Prune resolved items older than 30 days
   4. Cap resolved history at MAX_RESOLVED_HISTORY (keep newest by resolved_at)
   5. Write queue atomically
   6. Return (queue, "")

9. `get_pending_for_csv(csv_file: str) -> Tuple[List[Dict], str]` — Get all pending approval requests for a specific CSV file. Return (pending_items, error_msg).

10. `replay_approved_request(queue_item: Dict, session_key: str, admin_user: str) -> Tuple[bool, str]` — Replay an approved request's action against current state (via Phase 2 functions). Return (success, error_msg).

    Precondition validation:
    - If action_type not in _no_csv_actions: verify CSV file exists at path
    - If action_type in {add_rule, remove_rule, edit_rule}: verify rule exists in registry
    - Verify analyst still has permission to perform action (RBAC check)

    Replay logic:
    1. Extract action_type, payload from queue_item
    2. Call appropriate Phase 2 function with original analyst as context:
       - action_type="save_csv" → wl_csv.write_csv(path, headers, rows, analyst=original_analyst)
       - action_type="revert_csv" → wl_versions.snapshot_version(...) then wl_csv.write_csv(...)
       - etc.
    3. If replay succeeds: mark queue_item status="approved"
    4. If replay fails: mark queue_item status="failed", failure_reason=error_msg
    5. Build audit event: action="request_approved" or "request_failed", include replay result
    6. Post audit event
    7. Return (success, error_msg)

    _no_csv_actions = {create_csv, create_rule, remove_csv, remove_rule} (don't require CSV existence)

**Error Handling:**
- read_approval_queue: return empty list on file missing or JSON invalid (non-breaking)
- write_approval_queue: return (False, error_msg) on permission denied or write failure
- submit_approval_request: return ("", error_msg) on lock acquisition failure or write failure
- process_approval_decision: return (False, error_msg) on validation failure, lock acquisition failure, or replay failure
- All functions: no exceptions; all errors returned in tuples or logged

**Thread Safety:**
- All file operations protected by _approval_queue_lock (RLock allows reentrant acquisition)
- Approval decision processing is atomic via lock
