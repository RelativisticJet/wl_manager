**Simple POST actions in Wave 2 (6-8 handlers):**

1. `_action_save_col_widths()`:
   - Payload: {csv_file, column_widths: {col_name: width_px}}
   - Call wl_csv.set_column_widths(csv_file, column_widths)
   - Return {success: true}
   - Error handling: ValueError→400, IOError→500

2. `_action_mark_notifications_read()`:
   - Payload: {notification_ids: [...]} or empty (mark all)
   - Call wl_notify.mark_notifications_read(user, notification_ids)
   - Return {success: true, marked_count: int}
   - Error handling: standard exception mapping

3. `_action_cancel_request()`:
   - Payload: {request_id}
   - Call wl_approval.cancel_request(request_id, user)
   - Return {success: true}
   - Precondition: request_id must exist and belong to user
   - Error: FileNotFoundError→404, PermissionError→403

4. `_action_log_event()`:
   - Payload: {event_type, data}
   - Call wl_audit.post_audit_event(session_key, event)
   - Return {success: true}
   - Non-blocking: if audit fails, log but return success

5. `_action_save_as_default()`:
   - Payload: {key, value} (e.g., key="default_rule_csv", value="DR999.csv")
   - Parse config file, update key, write atomically
   - Return {success: true}
   - Keep simple file I/O inline (not worth a module)

6. `_action_reset_factory_defaults()`:
   - Payload: {} (no params needed)
   - Reset config file to factory defaults (empty or minimal)
   - Return {success: true}
   - Requires ADMIN_ROLES (check in dispatch table)

7. `_action_check_csv_status()` (if GET-like, move to GET_ACTIONS):
   - OR keep as POST if it's a write operation
   - Query CSV file status (exists, readable, writable)
   - Return {exists, readable, writable, size_bytes, mtime}
   - No side effects (read-only aggregation)

8. `_action_get_request_csv()` (read operation, may belong in GET_ACTIONS instead):
   - Query param: request_id
   - Fetch approval request by ID
   - Return {request_item: {...}}
   - Check auth: user must be requester or admin

**Handler signature:**
```python
def _action_X(self, request, payload, user, roles):
    try:
        # Validate payload
        field = payload.get("field")
        if not field:
            return self._resp(400, {"error": "field required"})
        
        # Call domain module or inline logic
        result = wl_*.function(...)
        
        # Return success response (with data if not None)
        return self._resp(200, {field: result, "success": True})
    except ValueError as e:
        return self._resp(400, {"error": str(e)})
    except FileNotFoundError as e:
        return self._resp(404, {"error": "Not found"})
    except PermissionError as e:
        return self._resp(403, {"error": "Permission denied"})
    except IOError as e:
        return self._resp(500, {"error": "File I/O error"})
    except Exception as e:
        _logger.exception(f"Error in {request.action}: {e}")
        return self._resp(500, {"error": "An internal error occurred"})
```

**Payload validation pattern:**
```python
def _action_X(self, request, payload, user, roles):
    # Validate required fields
    csv_file = payload.get("csv_file")
    if not csv_file:
        return self._resp(400, {"error": "csv_file required"})
    
    # Validate field values (e.g., is it a string, is it safe)
    if not isinstance(csv_file, str) or not csv_file:
        return self._resp(400, {"error": "csv_file must be non-empty string"})
    
    # Proceed with logic
```

**Success response pattern (no {success: true} for GET, only POST):**
```python
# GET response:
return self._resp(200, {csv_file, rows, columns})  # No success field

# POST response (simple stateless):
return self._resp(200, {success: True})  # Only success field, no data

# POST response (returns data):
return self._resp(200, {marked_count: 3, success: True})  # Data + success field
```
