## Locked Decisions from CONTEXT.md

### State Management (wl_state.js)
- **API:** `State.get(key)`, `State.set(key, val)`, `State.reset()`, `State.batch(updates)`, `State.isDirty()`, `State.on(event, callback)`, `State.off(event, callback)`, `State.register(key, defaultValue, validator)`
- **Fail-fast:** Unknown keys throw TypeError, validation failures throw TypeError
- **Events:** Setters fire jQuery custom events as `'state:keyName'` and `'state:reset'`
- **Computed property:** `isDirty()` auto-fires `'state:dirty'` event when currentRows !== originalRows
- **Batch mutations:** `State.batch({key: val, ...})` applies all at once, fires all events after
- **Shared keys:** currentRows, originalRows, selectedRows, detectionRuleSelected, csvFileSelected, pageIndex, columnWidths, isDirty (computed), pendingApprovalCount, adminPendingCount, userPresence, notificationCount

### REST Helpers (wl_rest.js)
- **Signature:** `restGet(action, params)` → jQuery promise with `.done(callback)` and `.fail(callback)`
- **Signature:** `restPost(action, payload)` → jQuery promise with `.done(callback)` and `.fail(callback)`
- **Error handling:** Default `.fail()` fires `'wl:restError'` event with {status, message, action}

### Module Communication
- **Custom events:** `wl:actionName` format for inter-module events on $(document)
- **State events:** `state:keyName` for state mutations

### Constants Module
- Extract all magic numbers, regex patterns, DOM selectors, config values into wl_constants.js
- Return object with SELECTORS, CONFIG, PATTERNS, role definitions, etc.

### UI Module (wl_ui.js)
- `showMsg(type, message)` — Shows toast/notification (error, success, warning, info)
- `showFatalError(message)` — Shows fatal error state, blocks interaction
- `toggleTheme()` — Switches between dark/light theme
- `init()` — Sets up message container, detects theme preference

### Notifications Module (rewrite)
- Rewrite as AMD module using `wl_rest.js` instead of direct $.ajax
- Replace `window.__wlNotifCallbacks` with `'wl:notificationsUpdated'` event
- Public API: `{init, start, stop}`

### Testing
- QUnit test framework, vendored in tests/qunit/
- Test files: test_state_manager.js, test_rest_helpers.js
- Hidden Splunk dashboard (test_runner.xml) loads QUnit + tests

### Build & Deployment
- Bump app.conf build number for cache busting
- Copy entire modules/ directory to container (wildcard)
- Clear Splunk i18n cache: `rm -f /opt/splunk/var/run/splunk/appserver/i18n/<filename>.js-*`
- Restart Splunk container
