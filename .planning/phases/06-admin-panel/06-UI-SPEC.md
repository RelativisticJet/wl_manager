# Phase 6 — UI Design Contract

> Visual and interaction contract for admin panel modularization. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (Splunk native + custom jQuery components) |
| Icon library | Splunk icon set (Feather icons) |
| Font | Splunk default (Roboto for modern theme, system fallback) |

**Rationale:** Phase 6 is a frontend modularization phase, not a redesign. The design system is inherited from Phase 5 and the existing Splunk ecosystem. No new design tools or libraries are introduced.

**Constraint:** jQuery + AMD modules only. No npm packages, no bundlers. AppInspect compliance required.

## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Compact element spacing (input borders, label margins) |
| md | 16px | Default element spacing, padding within modals |
| lg | 24px | Section padding, tab margins |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks (vertical spacing between queue/limits/usage tabs) |

**Exceptions:** None. All CP module layout must respect 8-point scale multiples.

**Implementation:** CSS custom properties in whitelist_manager.css (unchanged from Phase 5). CP modules use `.wl-*` prefixed classes that apply these values.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (normal) | 1.5 |
| Label | 13px | 600 (semibold) | 1.4 |
| Heading (queue title, section header) | 18px | 600 (semibold) | 1.3 |
| Display (page title "Control Panel") | 24px | 600 (semibold) | 1.2 |

**Rationale:**
- Body 14px matches Splunk web standard for form labels and table data
- Label 13px (semibold) for form field labels and table headers (differentiates from body text)
- Heading 18px for "Approval Queue", "Daily Limits", "Analyst Usage", "Trash" section titles
- Display 24px for the main "Control Panel" page title

**Exceptions:** None. All CP modules must use only these 4 sizes and 2 weights.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #fff (light) / #1a1c20 (dark) | Backgrounds for panels, main content areas |
| Secondary (30%) | #fafafa (light) / #111315 (dark) | Table rows, card backgrounds, sidebar sections |
| Accent (10%) | #1565c0 (blue) | Interactive elements (button hover, link color, tab underline) |
| Destructive | #c62828 (red) | Delete/purge actions, error states |
| Success | #2e7d32 (green) | Approve actions, success messages |
| Warning | #e65100 (orange) | Confirm-required actions, pending indicators |

**Accent Reserved For (SPECIFIC):**
- Approval queue: "Approve" button, pending request highlight on hover
- Daily limits: "Save" button, active form input border
- Usage table: Column sort indicator, selected row highlight
- Trash: "Restore" button, restore-on-hover highlight
- Admin limits: "Save" button, active field border
- Tab navigation: Active tab underline in Control Panel tabs

**DO NOT use accent for:** All interactive elements, general hover states, warning indicators. Accent is reserved for the specific interactive confirmations and state indicators listed above.

**Theme Support:**
- Light theme: Use hex values as defined
- Dark theme: CSS custom properties handled by `.wl-dark` class (applied by entry point via detectDarkTheme)
- No hardcoded theme values in CP modules; always use `var(--wl-bg)`, `var(--wl-text)`, etc.

## Component Structure

### Control Panel Entry Point (control_panel.js refactored)

**Visual Layout:**
```
┌─────────────────────────────────────────┐
│        Control Panel (Page Title)        │  24px heading
├─────────────────────────────────────────┤
│ [Queue] [Limits] [Usage] [Trash] [Admin]│  Tab navigation (underline active tab)
├─────────────────────────────────────────┤
│ (Tab content loaded via module.init())   │  Padding: 24px all sides (lg spacing)
│                                           │
└─────────────────────────────────────────┘
```

**Tab Interaction:**
- Clicking a tab calls `module.load()` for fresh data, then `module.startPolling()`
- URL updates via `history.replaceState(?tab=X)` for deep linking
- Pressing Escape on a modal does NOT switch tabs (modal has priority)
- Switching to active tab re-fetches data but doesn't interrupt current modal

**Polling Lifecycle:**
- Queue: Poll every 5 seconds when Queue tab is active. Pause when tab is hidden (via `document.visibilitychange`)
- Usage: Poll every 10 seconds when Usage tab is active. Pause when tab is hidden
- Limits: No polling (static form)
- Trash: No polling (list refreshes on action)
- Admin Limits: No polling (static form)

**Access Control UI:**
- If initial access check fails (403 Forbidden): Show single alert and return early (no tabs rendered)
- If module init throws: Show fatal error and stop (no partial CP initialization)
- Defense in depth: Entry point gates + module self-guards + server re-verification on destructive actions

## Interaction Patterns

### Modal Lifecycle
1. **Render:** Entry point creates hidden modals in DOM on first tab switch
2. **Show:** Module calls `ctx.showAlert()`, `ctx.showConfirm()`, or `ctx.showPrompt()`
3. **Dismiss:** User clicks button or presses Escape
4. **Cleanup:** Modal removed from DOM, event handlers cleaned up

### Form Validation
- Real-time validation on input change (show inline error or helper text update)
- Disable submit button if validation fails
- Show field-level error messages below input, red text (#c62828)

### Polling Lifecycle
1. **Tab becomes active:** Call `module.load()` for fresh data, then `module.startPolling()`
2. **Polling interval:** Queue 5s, Usage 10s (configurable constants)
3. **Tab becomes inactive:** Call `module.stopPolling()` to stop polling
4. **Browser hidden:** Stop all active polling via `document.visibilitychange`
5. **Browser visible again:** Refresh active module and resume polling

### Pagination
- Default page size: Queue=10, Usage=20, Trash=15 (configurable constants)
- Preserve page number when filtering (unless filtered results don't contain current page)
- Reset to page 1 when search text changes

### Keyboard Navigation
- Escape key: Dismiss open modal or close tooltip
- Tab: Navigate form fields and buttons (browser default)
- Enter in search field: Submit search (debounce handles rapid typing)
- Enter on button: Trigger action (default browser behavior)
