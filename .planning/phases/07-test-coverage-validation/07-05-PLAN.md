## Frontend Module Structure

- `appserver/static/whitelist_manager.js` (~1600 lines)
  - **wl_rest**: REST API wrapper (methods: GET `get_csv`, `get_mapping`, `get_versions`; POST `save_csv`, `revert_csv`)
  - **wl_table**: Data model (arrays: `originalRows`, `currentRows`, `deletedRows`; methods: `syncInputs()`, `refreshTable()`, `addRow()`, `deleteRow()`)
  - **wl_modals**: Modal lifecycle (modals: Create Rule, Remove Reason, Revert Reason, Search Conflict; methods: `openModal()`, `closeModal()`, `getModalValue()`)
  - **wl_state**: State coordination (calls wl_table, wl_modals, wl_rest in sequence)

## QUnit + Splunk Integration

- QUnit runs inside Splunk dashboard SimpleXML (test_runner.xml)
- MVC tokens available for data binding
- Mock Splunk REST client (MVC modules) to avoid real HTTP calls
- Test fixtures provided as inline JSON (sample CSV, approval queue)
- Browser automation via Playwright loads test_runner.xml and checks test results in DOM
